# 2. 数据策略与生态底盘

**Personal Data 是一切的底盘。** 它是多源聚合、持续积累、私有存储、可产权化、可交易、可交换的资产。

## 开放平台与通用接口 (Open Platform & Universal Interface)
Fidus 致力于构建一个提供 **相对规整、通用性访问接口** 的开放平台：
1.  **数据接口标准化**: 让不同的 Agent 都能以标准方式读取必要的 Context。
2.  **Personal Agent 生态**:
    *   **私有部署**: 安全运行在用户控制的环境中。
    *   **可组合/可进化**: Agent 之间可以嵌套调用，能力不断迭代。
    *   **社区共享**: 每个人都可以基于通用的数据接口，开发个性化、专有化的 Agent 以自用，或者分享给社区其他人用。

## 数据源的多源整合 (Data Sources)
Fidus 整合以下 8 类关键数据源：
1.  **个人体征与 Profile**: 特征、偏好、个性、风格等基础画像。
2.  **语音与对话**: 商务和日常的聊天、会议语音转写的文本。
3.  **Browser 轨迹**: 浏览器历史、搜索记录、页面访问。
4.  **App 轨迹**: 各类应用程序的使用记录和交互数据。
5.  **社交图谱**: 人脉网络、互动状态、链接强度、调动能力的参数集。
6.  **链上行为**: 区块链上的交易、持有与交互记录。
7.  **阅读笔记**: 书籍、文章、研报的阅读记录与笔记。
8.  **全终端行为**: 桌面端、移动端、AI 眼镜等界面可追踪的行为轨迹和知识。

## 数据用途 (Data Uses)
数据经处理后用于以下 7 个方面：
1.  **意图识别 + 细节填充**: 理解用户简短指令背后的完整需求。
2.  **框架性内容的缺口提醒**: 识别知识盲区或信息缺失并提醒。
3.  **特定需求的报表提取**: 自动生成个性化报表。
4.  **带单分享**: 基于社媒轨迹和投资行为的价值分享。
5.  **人脉价值测度**: 社交人脉资源的标签化、相关性分析和价值评估。
6.  **“可用不可见”分享**: 原始数据基于同态加密，实现分享和调用的结构化与接口标准化。
7.  **Agent 组合调用**: 支持 Agent 的组合化、嵌套式、可编程调用。

## 避免情况 (Avoidances)
1.  **规避大数据杀熟**: 防止供给端 Agent 或平台利用个人数据进行价格歧视。
2.  **避免迁移障碍**: 防止私密数据与特定 LLM 绑定，确保跨平台迁移能力。
