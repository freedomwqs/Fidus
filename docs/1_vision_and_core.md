# 1. 愿景与核心基座

## 产品定位
**Fidus** = **Personal Data & Private Agent**
属于你个人的数据和私人的代理。

## 核心目标
让每个人都成为 **AI-driven SuperMan**。

## 四大职能
1.  **贴身秘书**: 随时响应，处理繁杂事务。
2.  **忠实库管**: 整理、存储、索引个人数据资产。
3.  **智能代理**: 自主执行任务，代表用户利益。
4.  **极致效率**: 优化流程，最大化个人产出。

## 核心基座：私有知识库 + 私人 Agent

### 知识库特性
*   **横向打通**: 连接不同应用、平台的数据孤岛。
*   **纵向积累**: 随时间推移，数据深度和价值不断增加。
*   **结构化**: 将非结构化数据转化为可被 AI 理解的格式。
*   **死忠化**: 数据完全归属于用户，忠诚于用户利益。
*   **产权化**: 确立数据的所有权，为价值交换奠定基础。
*   **供需撮合与价值对接精准化**: 基于深度理解的精准匹配。
*   **知识和资源互换便利化**: 降低价值流动的门槛。

## 核心哲学：Context 闭包与数据底盘

> **"Personal Data 是 Context 的最小上界和最适配闭包 (量体裁衣、不多也不少)。"**

### 1. 智能的本质：高质量 Context
*   LLM 和 Agent 的核心能力都基于 **Context (上下文)**。
*   **Persona Data (个人数据)** 正是这个 Context 的**最小上界**和**最适配闭包**。
*   基于这种“量体裁衣”的 Context 输出的智能，才是最有质量、最贴心的用户侧智能。

### 2. 铁打的数据，流水的组件 (可替代性)
*   **可替代、可迁移、可插拔**:
    *   **LLM**: 随时可以被替代和互相 PK 的“外部高参”。
    *   **Agent**: 表现层的服务者，随时可能更迭。
    *   **App**: 功能载体，可随时替换。
*   **不可动摇的底盘**:
    *   **Personal Data**: 是一切的根基。它是多源聚合、持续积累、私有存储的资产。

### 3. 生态愿景：开放与赋能
*   **Personal Open Source**: 提倡个人数据的开放架构（非公开数据，而是开放接口）。
*   **通用数据接口**: Fidus 提供相对规整、通用性的访问接口。
*   **人人都是开发者**: 每个人都可以基于通用数据接口，开发个性化、专有化的 Agent 自用，或者分享给社区。

## 4. 命名由来与核心立场
> **"Fidus: Data Truly Yours, Agent Deeply Loyal. (数据私有，立场唯你)"**

### 词源 (Etymology)
*   **Fidus** = Latin `fides` (信任) + `-us` (值得信赖者) → **忠诚的代理**。

### 为什么值得信赖？
这种忠诚并非源于口号，而是源于**架构层面的根本性设计**：
*   **完全开源**: 代码透明，无黑箱操作。
*   **自己部署**: 运行在用户控制的本地或私有云环境。
*   **自己授权**: 只有用户拥有数据的最高权限。
*   **立场唯你**: 屁股完全站在用户侧，真正从你的角度出发帮你规划、对比和执行。

### 对比分析 (vs Platform Agents)
*   **Platform Agents (如 Manus)**: 数据和 Agent 都由平台收集、把控。用户的隐私是平台的资产。
*   **Fidus**: Agent 和数据都是个人的、私有的。用户的隐私是用户神圣不可侵犯的权利。
